# 修复 doxdocgen 中文乱码问题

## 问题分析

当前出现乱码的原因是：
```
-----@return ? s8
```

这是因为 `settings.json` 文件的编码问题。VS Code的配置文件应该使用 **UTF-8** 编码，但当前文件可能是 GB2312 编码。

## 解决方案

### 方案1：使用VS Code设置界面修改（推荐）

1. 打开 VS Code
2. 按 `Ctrl+Shift+P` 打开命令面板
3. 输入 `settings`，选择 **"首选项: 打开用户设置(JSON)"**
4. 在打开的 JSON 文件中，找到 doxdocgen 相关配置
5. 确保文件编码为 UTF-8：
   - 点击右下角编码显示（如 GB2312）
   - 选择 **"通过编码重新打开"** -> **UTF-8**
   - 如果提示保存，选择 **"保存并重新打开"**

6. 修改 `returnTemplate` 配置为：
   ```json
   "doxdocgen.generic.returnTemplate": "@return{indent:8}return {type}"
   ```
   注意：这里使用英文 "return" 而不是中文

7. 保存文件

### 方案2：直接修改文件编码

1. 在 VS Code 中打开 `c:/Users/29148/AppData/Roaming/Code/User/settings.json`
2. 点击右下角的编码显示（如 `GB2312`）
3. 选择 **"通过编码重新打开"** -> **UTF-8**
4. 如果中文显示正常，说明转换成功
5. 保存文件

### 方案3：使用英文标签（最简单）

如果不想处理编码问题，直接使用英文标签：

```json
"doxdocgen.generic.returnTemplate": "@return{indent:8}{type}"
```

生成的注释：
```c
----- @return s8
```

## 当前配置（已修复）

已将配置恢复为纯英文格式，避免编码问题：

```json
"doxdocgen.generic.paramTemplate": "@param{indent:8}{param}{indent:8}",
"doxdocgen.generic.returnTemplate": "@return{indent:8}{type}",
"doxdocgen.generic.dateTemplate": "@date{indent:8}{date}",
```

## 推荐做法

为了避免编码问题，建议：

1. **使用英文标签**：`@param`、`@return`、`@date`
2. **在@brief中写中文描述**：函数功能说明用中文
3. **参数描述用中文**：在参数行下方添加中文说明

这样既避免了编码问题，又能有中文文档。

## 完整示例

```c
/*********************************************************************************************************************
----- @brief 开关控制函数
----- 
----- 根据对象和类型执行开关操作。
----- 
----- @param[in] obj 开关对象
-----                  - SO_KEY:   按键控制
-----                  - SO_CONSOLE: 面板控制
-----                  - SO_PARA:   并机控制
----- @param[in] type 开关类型
-----                  - ST_NULL:  取反
-----                  - ST_ON:    打开
-----                  - ST_OFF:   关闭
----- @return s8 执行状态
-----         - 0: 成功
-----         - -1: 失败
----- 
----- @author LJD (291483914@qq.com)
----- @date 2026-01-09
******************************************************************************************************************/
s8 cBms_Switch(SwitchObject_E obj, SwitchType_E type);
```
