## 使用步骤

### 1. 生成基础注释

在函数上方按下 `/` 键（触发字符），doxdocgen会自动生成基础注释：

```c
/*********************************************************************************************************************
----- @brief 
----- @param power 
----- @param num 
----- @return void 
----- @author LJD (291483914@qq.com)
----- @date 2026-01-09
******************************************************************************************************************/
void Display_Num(uint16_t power, uint16_t num);
```

### 2. 手动添加输入输出标记

根据参数的实际用途，将 `@param` 修改为：
- `@param[in]` - 输入参数
- `@param[out]` - 输出参数
- `@param[in,out]` - 既输入又输出

修改后的完整注释：

```c
/*********************************************************************************************************************
----- @brief 显示数字和SOC百分比
----- 
----- 在LCD屏幕上显示数字值和对应的SOC百分比。
----- 
----- @param[in]  power 功率值（瓦特）
----- @param[in]  num   总功率值（用于计算SOC）
----- @return void
----- 
----- @author LJD (291483914@qq.com)
----- @date 2026-01-09
******************************************************************************************************************/
void Display_Num(uint16_t power, uint16_t num);
```

### 3. 参数命名建议

为了便于识别参数方向，建议遵循以下命名约定：

```c
// 输入参数：不使用特殊前缀
uint8_t data_soc;    // SOC值
uint16_t power;      // 功率值

// 输出参数：使用 p_ 前缀（pointer的缩写）
uint8_t *p_result;   // 结果指针
int *p_count;        // 计数指针

// 输入输出参数：使用 p_ 前缀
uint8_t *p_buffer;   // 缓冲区指针（既输入又输出）
```

## 快速修改技巧

### 批量修改参数方向

1. 生成注释后，使用查找替换功能：
   - 查找：`@param power`
   - 替换为：`@param[in] power`

2. 对于输出参数：
   - 查找：`@param p_result`
   - 替换为：`@param[out] p_result`

### 使用代码片段加速

创建VS Code代码片段（用户代码片段 > C.json）：

```json
{
    "Doxygen In Param": {
        "prefix": "paramin",
        "description": "添加输入参数注释",
        "body": ["@param[in] $1 $2"]
    },
    "Doxygen Out Param": {
        "prefix": "paramout",
        "description": "添加输出参数注释",
        "body": ["@param[out] $1 $2"]
    },
    "Doxygen InOut Param": {
        "prefix": "paraminout",
        "description": "添加输入输出参数注释",
        "body": ["@param[in,out] $1 $2"]
    }
}
```

## 常见问题

### Q1: 为什么doxdocgen不能自动识别输入输出参数？

**A:** doxdocgen插件本身不会自动分析参数的使用方向，需要开发者根据函数的实际功能手动添加 `[in]`、`[out]` 标记。这是Doxygen的标准做法。

### Q2: 生成的注释中没有 `@param` 项？

**A:** 请检查：
1. 是否在函数声明上方生成注释（而不是函数实现）
2. 配置中是否添加了 `doxdocgen.generic.paramTemplate`
3. 函数是否有参数（无参数函数不会生成 `@param`）

### Q3: 如何让doxdocgen识别我的函数？

**A:** 确保满足以下条件：
- 在函数声明的上方按 `/` 键
- 函数声明格式正确（有参数列表）
- 配置中 `doxdocgen.generic.linesToGet` 设置足够大（当前设置为20）

### Q4: 生成的注释格式不对？

**A:** 检查配置中的以下项：
```json
"doxdocgen.c.commentPrefix": "----- ",
"doxdocgen.c.firstLine": "/*****************************************************************************************************************",
"doxdocgen.c.lastLine": "******************************************************************************************************************/"
```

## 最佳实践

1. **一致性**：在整个项目中保持参数命名和注释风格一致
2. **清晰性**：参数描述应简明扼要，说明参数的含义和范围
3. **完整性**：所有参数都应有对应的注释
4. **可读性**：合理使用空行和缩进，使注释易于阅读
